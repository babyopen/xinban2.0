# 启动应用与功能测试指南

## 一、启动应用

### 1. 使用宝塔启动脚本

**步骤**：

1. 登录服务器
2. 进入项目目录：
  ```bash
   cd /www/wwwroot/xboyi.cn
  ```
3. 执行启动脚本：
  ```bash
   ./baota_start.sh
  ```
4. 等待启动完成，查看启动日志

### 2. 使用 Python项目管理器

**步骤**：

1. 登录宝塔面板
2. 进入 `Python项目管理器`
3. 找到 `xboyi.cn` 项目
4. 点击 `启动` 按钮
5. 查看启动状态

### 3. 检查应用状态

**步骤**：

1. 执行：
  ```bash
   # 检查进程状态
   ps aux | grep gunicorn | grep api.index:app

   # 检查端口占用
   lsof -i :8000

   # 检查日志
   tail -f logs/gunicorn_error.log
  ```

## 四、常见问题

### 1. 应用启动失败

**原因**：

- 依赖未安装
- 端口被占用
- 模型文件缺失
- 配置文件错误

**解决**：

- 检查依赖安装情况
- 检查端口占用情况
- 检查模型文件是否存在
- 检查配置文件是否正确

### 2. API 响应错误

**原因**：

- 应用未启动
- 路由配置错误
- 代码逻辑错误
- 数据格式错误

**解决**：

- 检查应用状态
- 检查路由配置
- 查看应用日志
- 检查请求数据格式

### 3. 页面显示异常

**原因**：

- 静态文件未加载
- 样式文件路径错误
- JavaScript 错误
- 网络连接问题

**解决**：

- 检查静态文件路径
- 检查样式文件加载
- 查看浏览器控制台错误
- 检查网络连接

### 4. 性能问题

**原因**：

- 服务器资源不足
- 代码优化不足
- 数据库查询慢
- 缓存配置不当

**解决**：

- 增加服务器资源
- 优化代码逻辑
- 优化数据库查询
- 配置缓存策略

## 五、故障排查

### 1. 查看日志

**步骤**：

1. **应用日志**：
  ```bash
   tail -f logs/gunicorn_error.log
   tail -f logs/gunicorn_access.log
  ```
2. **Nginx 日志**：
  ```bash
   tail -f /www/wwwlogs/xboyi.cn.error.log
   tail -f /www/wwwlogs/xboyi.cn.access.log
  ```
3. **系统日志**：
  ```bash
   tail -f /var/log/messages
   tail -f /var/log/syslog
  ```

### 2. 检查服务状态

**步骤**：

1. **检查 Nginx**：
  ```bash
   systemctl status nginx
   nginx -t
  ```
2. **检查应用进程**：
  ```bash
   ps aux | grep gunicorn
   netstat -tuln | grep 8000
  ```
3. **检查网络连接**：
  ```bash
   ping www.xboyi.cn
   curl -I http://www.xboyi.cn
  ```

### 3. 重启服务

**步骤**：

1. **重启应用**：
  ```bash
   # 停止应用
   ps aux | grep gunicorn | grep api.index:app | grep -v grep | awk '{print $2}' | xargs kill -9

   # 启动应用
   ./baota_start.sh
  ```
2. **重启 Nginx**：
  ```bash
   systemctl restart nginx
  ```
3. **重启服务器**（最后手段）：
  ```bash
   reboot
  ```
