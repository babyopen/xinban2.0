# 文件权限与安全设置指南

## 一、文件权限配置

### 1. 项目文件权限

**步骤**：
1. 登录服务器
2. 进入项目目录：
   ```bash
   cd /www/wwwroot/xboyi.cn
   ```
3. 设置文件权限：
   ```bash
   # 设置所有文件权限为 644
   find . -type f -exec chmod 644 {} \;
   
   # 设置所有目录权限为 755
   find . -type d -exec chmod 755 {} \;
   
   # 设置脚本文件执行权限
   chmod +x baota_start.sh
   chmod +x start.sh
   ```

### 2. 日志目录权限

**步骤**：
1. 执行：
   ```bash
   # 创建日志目录（如果不存在）
   mkdir -p logs
   
   # 设置日志目录权限
   chmod 775 logs/
   chmod g+s logs/
   ```

### 3. 临时目录权限

**步骤**：
1. 执行：
   ```bash
   # 创建临时目录（如果不存在）
   mkdir -p temp
   
   # 设置临时目录权限
   chmod 775 temp/
   chmod g+s temp/
   ```

### 4. 模型和数据文件权限

**步骤**：
1. 执行：
   ```bash
   # 设置模型文件权限
   chmod 644 zodiac_model.pkl
   
   # 设置数据文件权限
   chmod 644 lottery_history.csv
   ```

## 二、用户权限配置

### 1. 创建专用用户

**步骤**：
1. 执行：
   ```bash
   # 创建用户
   useradd -m xboyi
   
   # 设置密码
   passwd xboyi
   
   # 添加到 www 组
   usermod -aG www xboyi
   ```

### 2. 修改文件所有者

**步骤**：
1. 执行：
   ```bash
   # 修改项目目录所有者
   chown -R xboyi:www /www/wwwroot/xboyi.cn
   
   # 特别设置日志和临时目录
   chown -R xboyi:www /www/wwwroot/xboyi.cn/logs
   chown -R xboyi:www /www/wwwroot/xboyi.cn/temp
   ```

### 3. 限制敏感文件访问

**步骤**：
1. 执行：
   ```bash
   # 限制配置文件访问
   chmod 600 gunicorn_config.py
   chmod 600 nginx_config.conf
   ```

## 三、防火墙配置

### 1. 检查防火墙状态

**步骤**：
1. 执行：
   ```bash
   # CentOS 系统
   systemctl status firewalld
   
   # Ubuntu/Debian 系统
   ufw status
   ```

### 2. 开放必要端口

**步骤**：
1. 执行：
   ```bash
   # CentOS 系统
   firewall-cmd --zone=public --add-port=80/tcp --permanent
   firewall-cmd --zone=public --add-port=443/tcp --permanent
   firewall-cmd --zone=public --add-port=8000/tcp --permanent
   firewall-cmd --reload
   
   # Ubuntu/Debian 系统
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw allow 8000/tcp
   ufw reload
   ```

### 3. 关闭不必要端口

**步骤**：
1. 执行：
   ```bash
   # 查看已开放端口
   # CentOS 系统
   firewall-cmd --list-ports
   
   # Ubuntu/Debian 系统
   ufw status
   
   # 关闭不必要的端口
   # CentOS 系统
   firewall-cmd --zone=public --remove-port=端口号/tcp --permanent
   firewall-cmd --reload
   
   # Ubuntu/Debian 系统
   ufw deny 端口号/tcp
   ```

## 四、安全加固

### 1. SSH 安全配置

**步骤**：
1. 编辑 SSH 配置文件：
   ```bash
   vim /etc/ssh/sshd_config
   ```
2. 修改以下配置：
   ```
   # 修改 SSH 端口
   Port 2222
   
   # 禁用 root 登录
   PermitRootLogin no
   
   # 禁用密码登录（使用密钥登录）
   PasswordAuthentication no
   
   # 限制登录尝试次数
   MaxAuthTries 3
   
   # 限制登录时间
   LoginGraceTime 30
   ```
3. 重启 SSH 服务：
   ```bash
   # CentOS 系统
   systemctl restart sshd
   
   # Ubuntu/Debian 系统
   systemctl restart ssh
   ```

### 2. 安装 Fail2ban

**步骤**：
1. 执行：
   ```bash
   # CentOS 系统
   yum install -y epel-release
   yum install -y fail2ban
   
   # Ubuntu/Debian 系统
   apt install -y fail2ban
   ```
2. 配置 Fail2ban：
   ```bash
   vim /etc/fail2ban/jail.local
   ```
3. 添加配置：
   ```
   [sshd]
enabled = true
port = 2222
filter = sshd
logpath = /var/log/auth.log
maxretry = 5
bantime = 3600

[nginx-http-auth]
enabled = true
filter = nginx-http-auth
logpath = /www/wwwlogs/xboyi.cn.error.log
maxretry = 3
bantime = 3600
   ```
4. 启动 Fail2ban：
   ```bash
   # CentOS 系统
   systemctl start fail2ban
   systemctl enable fail2ban
   
   # Ubuntu/Debian 系统
   systemctl start fail2ban
   systemctl enable fail2ban
   ```

### 3. 安装并配置 ModSecurity

**步骤**：
1. 通过宝塔面板安装 ModSecurity：
   - 进入 `软件商店`
   - 找到 `Nginx`
   - 点击 `设置`
   - 进入 `模块管理`
   - 安装 `ModSecurity` 模块

2. 配置 ModSecurity：
   - 启用 OWASP 核心规则集
   - 调整规则以减少误报

### 4. 定期更新系统

**步骤**：
1. 执行：
   ```bash
   # CentOS 系统
   yum update -y
   
   # Ubuntu/Debian 系统
   apt update && apt upgrade -y
   ```

## 五、应用安全

### 1. 配置 Gunicorn 安全选项

**步骤**：
1. 编辑 `gunicorn_config.py`：
   ```python
   # 设置工作进程数
   workers = 3
   
   # 设置线程数
   threads = 2
   
   # 绑定地址
   bind = '127.0.0.1:8000'
   
   # 超时设置
   timeout = 60
   
   # 日志配置
   accesslog = 'logs/gunicorn_access.log'
   errorlog = 'logs/gunicorn_error.log'
   loglevel = 'info'
   
   # 安全选项
   limit_request_line = 4094
   limit_request_fields = 100
   limit_request_field_size = 8190
   ```

### 2. 配置 Flask 安全选项

**步骤**：
1. 编辑 `api/index.py`，确保以下安全配置：
   ```python
   # 禁用调试模式
   app.debug = False
   
   # 配置会话密钥
   app.secret_key = os.urandom(24)
   
   # 配置内容安全策略
   @app.after_request
   def add_security_headers(response):
       response.headers['X-Content-Type-Options'] = 'nosniff'
       response.headers['X-Frame-Options'] = 'SAMEORIGIN'
       response.headers['X-XSS-Protection'] = '1; mode=block'
       return response
   ```

### 3. 输入验证

**步骤**：
1. 确保所有用户输入都经过验证：
   - 验证 API 请求参数
   - 验证文件上传
   - 防止 SQL 注入
   - 防止 XSS 攻击

## 六、监控与审计

### 1. 配置日志监控

**步骤**：
1. 执行：
   ```bash
   # 配置日志轮转
   vim /etc/logrotate.d/xboyi
   ```
2. 添加配置：
   ```
   /www/wwwroot/xboyi.cn/logs/*.log {
       daily
       rotate 7
       compress
       delaycompress
       missingok
       notifempty
       create 644 xboyi www
   }
   ```

### 2. 安装监控工具

**步骤**：
1. 通过宝塔面板安装监控工具：
   - 进入 `软件商店`
   - 搜索并安装 `宝塔监控`
   - 配置监控项：CPU、内存、磁盘、网络

### 3. 定期安全扫描

**步骤**：
1. 执行：
   ```bash
   # 安装安全扫描工具
   # CentOS 系统
   yum install -y rkhunter chkrootkit
   
   # Ubuntu/Debian 系统
   apt install -y rkhunter chkrootkit
   
   # 运行安全扫描
   rkhunter --check
   chkrootkit
   ```

## 七、常见问题

### 1. 权限错误

**原因**：
- 文件权限设置不正确
- 用户权限不足
- 目录所有权错误

**解决**：
- 重新设置文件权限
- 确保用户有足够的权限
- 检查目录所有权

### 2. 防火墙阻止访问

**原因**：
- 端口未开放
- 防火墙规则错误
- 网络连接问题

**解决**：
- 开放必要的端口
- 检查防火墙规则
- 测试网络连接

### 3. SSH 连接失败

**原因**：
- SSH 端口未开放
- SSH 配置错误
- 密钥认证失败

**解决**：
- 开放 SSH 端口
- 检查 SSH 配置
- 验证密钥配置

### 4. 安全漏洞

**原因**：
- 系统未及时更新
- 应用依赖存在漏洞
- 配置不当

**解决**：
- 定期更新系统
- 更新依赖包
- 检查并修复配置

## 八、最佳实践

1. **最小权限原则**：只授予必要的权限
2. **定期备份**：定期备份项目文件和数据
3. **安全更新**：及时更新系统和依赖
4. **监控日志**：定期检查日志文件
5. **安全扫描**：定期进行安全扫描
6. **使用 HTTPS**：配置 SSL 证书
7. **限制访问**：使用防火墙和访问控制
8. **审计日志**：记录关键操作

## 九、总结

文件权限和安全设置是项目部署中不可忽视的重要环节。通过合理配置文件权限、用户权限、防火墙和安全加固措施，可以有效提高系统的安全性和稳定性。

安全配置应遵循最小权限原则，只授予必要的权限，并定期进行安全检查和更新。同时，通过监控和审计，可以及时发现和处理安全问题。

在部署过程中，应始终将安全放在首位，确保应用和服务器的安全运行。
