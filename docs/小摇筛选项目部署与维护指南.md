# 小摇筛选项目部署与维护指南

## 一、项目概述

### 1. 项目简介

小摇筛选项目是一个基于 Flask 和机器学习的生肖预测系统，主要功能包括：
- 生肖预测：基于历史数据和机器学习模型预测下一期生肖
- 数据展示：展示预测结果和相关数据
- API 接口：提供 RESTful API 接口供其他系统调用

### 2. 技术栈

| 类别 | 技术 | 版本 |
|------|------|------|
| 后端框架 | Flask | 2.0.1 |
| 生产服务器 | Gunicorn | 20.1.0 |
| 数据处理 | Pandas | 1.3.3 |
| 机器学习 | scikit-learn | 0.24.2 |
| Web 服务器 | Nginx | 1.18+ |
| 前端 | HTML5 + CSS3 + JavaScript | - |

## 二、系统要求

### 1. 服务器要求

- **操作系统**：CentOS 7/8 或 Ubuntu 18.04+
- **内存**：至少 2GB
- **CPU**：至少 1核
- **磁盘**：至少 20GB 空闲空间
- **网络**：公网IP，开放 80/443 端口

### 2. 软件要求

- **Python**：3.8 或更高
- **Nginx**：1.18 或更高
- **宝塔面板**：6.0 或更高

## 三、部署步骤

### 1. 环境准备

**步骤**：
1. **安装宝塔面板**：
   ```bash
   # CentOS 系统
   yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
   
   # Ubuntu/Debian 系统
   wget -O install.sh http://download.bt.cn/install/install-ubuntu_6.0.sh && sudo bash install.sh
   ```

2. **安装必要软件**：
   - 登录宝塔面板
   - 进入 `软件商店`
   - 安装 `Nginx`
   - 安装 `Python项目管理器`

3. **域名解析**：
   - 在域名管理平台将 `www.xboyi.cn` 解析到服务器公网IP

### 2. 项目部署

**步骤**：
1. **上传项目文件**：
   - 进入 `/www/wwwroot/` 目录
   - 创建 `xboyi.cn` 目录
   - 上传项目所有文件到该目录

2. **配置 Python 环境**：
   - 打开 `Python项目管理器`
   - 点击 `添加项目`
   - 配置：
     - 项目路径：`/www/wwwroot/xboyi.cn`
     - Python版本：3.8 或更高
     - 框架：Flask
     - 启动方式：Gunicorn
     - 端口：8000
   - 点击 `确定`
   - 进入项目 `设置` → `模块`，安装依赖

3. **配置 Nginx**：
   - 进入 `网站` → `添加站点`
   - 配置：
     - 域名：`www.xboyi.cn`
     - 根目录：`/www/wwwroot/xboyi.cn/frontend`
     - PHP版本：纯静态
   - 点击 `提交`
   - 进入 `设置` → `反向代理`
   - 点击 `添加反向代理`
   - 配置：
     - 代理名称：`flask_app`
     - 目标URL：`http://127.0.0.1:8000`
   - 点击 `提交`
   - 进入 `配置文件`，替换为项目中的 `nginx_config.conf` 内容

4. **配置文件权限**：
   ```bash
   cd /www/wwwroot/xboyi.cn
   chmod -R 755 .
   chmod +x baota_start.sh
   mkdir -p logs temp
   chmod 775 logs/ temp/
   ```

5. **启动应用**：
   ```bash
   ./baota_start.sh
   ```

### 3. 功能测试

**步骤**：
1. **基本访问**：
   - 访问 `http://www.xboyi.cn`
   - 检查页面是否正常加载

2. **API 测试**：
   - 健康检查：`http://www.xboyi.cn/api/health`
   - 生肖映射：`http://www.xboyi.cn/api/zodiac-mapping`
   - 预测接口：使用 POST 请求测试

3. **性能测试**：
   - 检查页面加载时间
   - 测试 API 响应时间

## 四、维护指南

### 1. 日常维护

**任务**：
1. **日志检查**：
   - 定期检查应用日志：`logs/gunicorn_error.log`
   - 定期检查 Nginx 日志：`/www/wwwlogs/xboyi.cn.error.log`

2. **备份**：
   - 定期备份项目文件：
     ```bash
     cd /www/wwwroot
     tar -czf xboyi_backup_$(date +%Y%m%d).tar.gz xboyi.cn/
     ```
   - 定期备份模型文件和数据文件

3. **更新**：
   - 定期更新系统：
     ```bash
     # CentOS 系统
     yum update -y
     
     # Ubuntu/Debian 系统
     apt update && apt upgrade -y
     ```
   - 定期更新依赖：
     ```bash
     cd /www/wwwroot/xboyi.cn
     pip3 install -r requirements.txt --upgrade
     ```

### 2. 故障处理

**常见问题**：
1. **应用无法启动**：
   - 检查依赖是否安装
   - 检查端口是否被占用
   - 检查模型文件是否存在

2. **API 响应错误**：
   - 检查应用是否运行
   - 查看应用日志
   - 检查请求数据格式

3. **页面显示异常**：
   - 检查静态文件路径
   - 检查样式文件加载
   - 查看浏览器控制台错误

4. **性能问题**：
   - 检查服务器资源使用情况
   - 优化代码逻辑
   - 配置缓存策略

### 3. 安全维护

**措施**：
1. **定期安全扫描**：
   ```bash
   # 安装安全扫描工具
   yum install -y rkhunter chkrootkit
   
   # 运行安全扫描
   rkhunter --check
   chkrootkit
   ```

2. **更新 SSL 证书**：
   - 定期更新 Let's Encrypt 证书
   - 确保证书有效性

3. **防火墙配置**：
   - 定期检查防火墙规则
   - 关闭不必要的端口

4. **访问控制**：
   - 限制 SSH 登录
   - 使用密钥认证
   - 定期更换密码

## 七、常见问题与解决方案

### 1. 依赖安装失败

**原因**：
- 网络连接不稳定
- Python版本不兼容
- 缺少系统依赖

**解决**：
- 使用国内镜像源：
  ```bash
  pip3 install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
  ```
- 安装系统依赖：
  ```bash
  # CentOS 系统
  yum install -y gcc gcc-c++ make openssl-devel libffi-devel
  ```

### 2. 应用启动失败

**原因**：
- 端口被占用
- 模型文件缺失
- 配置文件错误

**解决**：
- 检查端口占用：
  ```bash
  lsof -i :8000
  ```
- 检查模型文件：
  ```bash
  ls -lh zodiac_model.pkl
  ```
- 查看应用日志：
  ```bash
  tail -f logs/gunicorn_error.log
  ```

### 3. 页面显示异常

**原因**：
- 静态文件未加载
- 样式文件路径错误
- Nginx 配置错误

**解决**：
- 检查静态文件路径：
  ```bash
  ls -la frontend/
  ```
- 检查 Nginx 配置：
  ```bash
  nginx -t
  ```
- 重启 Nginx：
  ```bash
  systemctl restart nginx
  ```

### 4. API 响应错误

**原因**：
- 应用未启动
- 路由配置错误
- 数据格式错误

**解决**：
- 检查应用状态：
  ```bash
  ps aux | grep gunicorn | grep api.index:app
  ```
- 检查路由配置：查看 `api/index.py`
- 检查请求数据格式
