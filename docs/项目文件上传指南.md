# 项目文件上传指南

## 一、上传前准备

### 1. 项目文件清单

确保以下文件和目录已准备就绪：

| 文件/目录 | 说明 | 必要性 |
|----------|------|--------|
| `api/` | 后端API代码 | 必需 |
| `frontend/` | 前端页面代码 | 必需 |
| `python/` | Python模块 | 必需 |
| `zodiac_model.pkl` | 预测模型文件 | 必需 |
| `lottery_history.csv` | 历史数据文件 | 必需 |
| `requirements.txt` | 依赖文件 | 必需 |
| `gunicorn_config.py` | Gunicorn配置 | 必需 |
| `nginx_config.conf` | Nginx配置 | 必需 |
| `baota_start.sh` | 宝塔启动脚本 | 必需 |
| `宝塔部署详细指南.md` | 部署文档 | 可选 |
| `域名解析与服务器配置指南.md` | 配置文档 | 可选 |

### 2. 服务器目录结构

宝塔服务器上的目标目录结构：

```
/www/wwwroot/xboyi.cn/
├── api/               # 后端API代码
├── frontend/          # 前端页面代码
├── python/            # Python模块
├── logs/              # 日志目录（自动创建）
├── temp/              # 临时目录（自动创建）
├── zodiac_model.pkl   # 预测模型文件
├── lottery_history.csv # 历史数据文件
├── requirements.txt   # 依赖文件
├── gunicorn_config.py # Gunicorn配置
├── nginx_config.conf  # Nginx配置
└── baota_start.sh     # 宝塔启动脚本
```

## 二、上传方法

### 方法一：使用宝塔面板文件管理

**步骤**：
1. 登录宝塔面板
2. 进入 `文件` 管理
3. 进入 `/www/wwwroot/` 目录
4. 点击 `新建目录`，创建 `xboyi.cn` 目录
5. 进入 `xboyi.cn` 目录
6. 点击 `上传` 按钮
7. 选择项目文件，点击 `开始上传`
8. 上传完成后，检查文件是否完整

**优点**：
- 操作简单，不需要命令行
- 支持批量上传
- 可视化操作界面

**缺点**：
- 大文件上传速度较慢
- 可能受浏览器限制

### 方法二：使用 SFTP 客户端

**推荐工具**：
- WinSCP（Windows）
- FileZilla（跨平台）
- Cyberduck（Mac）

**步骤**：
1. 打开 SFTP 客户端
2. 配置连接信息：
   - 主机：服务器公网IP
   - 端口：22（或修改后的SSH端口）
   - 用户名：root 或创建的普通用户
   - 密码：对应用户的密码
3. 连接服务器
4. 本地浏览到项目目录
5. 远程浏览到 `/www/wwwroot/` 目录
6. 创建 `xboyi.cn` 目录
7. 将项目文件拖拽到远程 `xboyi.cn` 目录
8. 等待上传完成

**优点**：
- 支持断点续传
- 上传速度较快
- 支持文件夹批量上传

**缺点**：
- 需要安装客户端软件
- 配置稍复杂

### 方法三：使用 Git 克隆

**步骤**：
1. 登录服务器
2. 进入 `/www/wwwroot/` 目录
3. 执行：
   ```bash
   git clone 项目仓库地址 xboyi.cn
   ```
4. 等待克隆完成

**优点**：
- 操作简单快捷
- 便于后续更新
- 自动保持文件结构

**缺点**：
- 需要项目已托管到Git仓库
- 可能需要配置SSH密钥

### 方法四：使用 scp 命令

**步骤**：
1. 在本地终端执行：
   ```bash
   # 上传整个项目目录
   scp -r 项目目录 root@服务器公网IP:/www/wwwroot/xboyi.cn/
   
   # 或上传单个文件
   scp 文件名 root@服务器公网IP:/www/wwwroot/xboyi.cn/
   ```
2. 输入服务器密码
3. 等待上传完成

**优点**：
- 命令行操作，适合批量上传
- 无需安装额外软件
- 支持压缩传输

**缺点**：
- 需要熟悉命令行
- 大文件上传速度可能较慢

## 三、上传后验证

### 1. 检查文件完整性

**步骤**：
1. 登录服务器
2. 进入 `/www/wwwroot/xboyi.cn` 目录
3. 执行：
   ```bash
   ls -la
   ```
4. 检查所有文件是否存在
5. 检查目录结构是否正确

### 2. 检查文件权限

**步骤**：
1. 执行：
   ```bash
   ls -la
   ```
2. 确保文件权限正确（通常为 644）
3. 确保目录权限正确（通常为 755）
4. 确保脚本文件有执行权限

### 3. 检查关键文件

**步骤**：
1. 检查模型文件：
   ```bash
   ls -lh zodiac_model.pkl
   ```
2. 检查历史数据文件：
   ```bash
   ls -lh lottery_history.csv
   ```
3. 检查依赖文件：
   ```bash
   cat requirements.txt
   ```

## 四、常见问题

### 1. 上传失败

**原因**：
- 网络连接不稳定
- 服务器磁盘空间不足
- 权限不足

**解决**：
- 检查网络连接
- 检查服务器磁盘空间：`df -h`
- 检查目录权限：`ls -la /www/wwwroot/`

### 2. 文件缺失

**原因**：
- 上传过程中断
- 文件被防火墙拦截
- 上传路径错误

**解决**：
- 重新上传缺失的文件
- 检查服务器防火墙设置
- 确认上传路径正确

### 3. 权限错误

**原因**：
- 文件权限设置不正确
- 目录权限设置不正确
- 用户权限不足

**解决**：
- 设置正确的文件权限：`chmod 644 文件名`
- 设置正确的目录权限：`chmod 755 目录名`
- 确保用户有足够的权限

### 4. 上传速度慢

**原因**：
- 网络带宽限制
- 文件体积过大
- 服务器负载过高

**解决**：
- 使用压缩后上传
- 选择网络较好的时间段
- 检查服务器负载：`top`

## 五、最佳实践

1. **备份原文件**：上传前备份服务器上的原有文件
2. **分批上传**：大文件分批上传，避免中断
3. **验证文件**：上传后验证文件完整性和权限
4. **记录日志**：记录上传过程，便于排查问题
5. **使用版本控制**：推荐使用Git进行版本管理

## 六、总结

选择合适的上传方法，确保项目文件完整、正确地上传到服务器。上传完成后，进行必要的验证，确保所有文件都已就绪，为后续的环境配置和应用启动做好准备。
